# YOKONEX Play DEMO · UI 界面设计

> 对应 Excalidraw 线框图：`ui_flow.excalidraw`

---

## 1. 页面路由

```
/                    → 首页看板（HomePage）
/chat               → 对话设计（ChatPage）
/preview            → 方案预览（PreviewPage）
/execution          → 执行中（ExecutionPage）
/library            → 玩法库（LibraryPage）
/connect            → 蓝牙连接（ConnectPage）
```

## 3. 对话设计（ChatPage）

### 布局

```
┌─────────────────────────────┐
│ ←  玩法设计                  │  ← 顶部导航栏
├─────────────────────────────┤
│ 🔒💧⚡🌀  全部在线         │  ← 玩具状态条（水平滚动）
├─────────────────────────────┤
│                              │
│ ┌─────────────────────────┐ │
│ │ 欢迎来到玩法设计！已检测 │ │  ← Agent 消息气泡
│ │ 到4个玩具。先选基础——你 │ │     · 紫色底色（#e5dbff）
│ │ 想用哪几个玩具？        │ │     · 居左
│ │  推荐：全部使用         │ │
│ │  A: 全部使用   B: 部分  │ │
│ └─────────────────────────┘ │
│                              │
│        ┌─────────────────┐  │
│        │ 全部使用，5分钟  │  │  ← 用户消息气泡
│        │ 渐进式          │  │     · 蓝色底色（#dbe4ff）
│        └─────────────────┘  │     · 居右
│                              │
│ ┌─────────────────────────┐ │
│ │ 好的！那节奏模式呢？   │ │  ← 下一轮 Agent 回复
│ │ 推荐：混合节奏 (前期    │ │
│ │ 舒缓，中期起伏，后期   │ │
│ │ 密集冲刺)              │ │
│ └─────────────────────────┘ │
│                              │
│          ...（滚动查看更多）  │
│                              │
├─────────────────────────────┤
│ ┌───────────────────┐ ➤ │  ← 输入框 + 发送按钮
│ │ 输入玩法描述...    │   │
│ └───────────────────┘   │
└─────────────────────────────┘
```

### 组件清单

| 组件 | 说明 |
|------|------|
| ToyStatusStrip | 水平可滚动的玩具状态标签条 |
| AgentBubble | 紫色左对齐气泡，包含文字+推荐选项 |
| UserBubble | 蓝色右对齐气泡，用户文字 |
| ChatInput | 底部输入栏 + 发送按钮 |
| OptionChips | Agent 消息里的推荐选项（可选 Chip 样式） |

### 关键交互

1. **Agent 主动提问**：每次发一条消息，包含推荐选项
2. **用户选选项**：点击 Chip 快速回复，也可打字输入
3. **对话结束时**：Agent 最后一条消息显示「预览方案」按钮
4. **加载状态**：Agent 生成回复时显示打字中动画（...）

### 状态

| 状态 | 表现 |
|------|------|
| 空闲 | 显示输入框 |
| Agent 生成中 | 显示「💭 正在设计方案...」+ 禁用输入 |
| 方案就绪 | 最后一条消息含「📜 预览方案」内联按钮 |

```
┌─────────────────────────────┐
│ ←  方案预览                  │

├─────────────────────────────┤
│ 步骤时间轴                   │
│ ┌─────────────────────────┐ │
│ │ ① 开始    绑定 + 启动  │ │  ← 绿色（已完成状态）
│ ├─────────────────────────┤ │
│ │ ② 等待    夹紧达阈值   │ │  ← 黄色（当前等待）
│ ├─────────────────────────┤ │
│ │ ③ 0-30s  赎罪计时      │ │  ← 橙色（即将到来）
│ ├─────────────────────────┤ │
│ │ ④ 30-90s ⛰️ 电击惩罚  │ │  ← 红色（高潮阶段）
│ ├─────────────────────────┤ │
│ │ ⑤ 90s    解锁 · 停止  │ │  ← 绿色（结束）
│ └─────────────────────────┘ │
│                              │
│ ┌─── Lua 脚本 ────────────┐ │
│ │ -- 玩法：【枷锁回响】   │ │  ← 可折叠区域
│ │ toy.lock_1:lock()       │ │     · 默认折叠
│ │ toy.mast_1:rotate(40)  │ │     · 点击展开全文
│ │ ...                     │ │     · 等宽字体
│ └─────────────────────────┘ │
│                              │
│  ┌────────────┐ ┌────────┐  │
│  │ ▶  执行    │ │ 重新生 │  │  ← 操作按钮
│  └────────────┘ │ 成    │  │
│                └────────┘  │
└─────────────────────────────┘
```

### 组件清单

| 组件 | 说明 |
|------|------|
| PlaybookName | 大号玩法标题 |
| DurationBadge | 时长标签 |
| StepTimeline | 纵向时间轴，每步有色块+时间+文字 |
| CodeBlock | 可折叠的 Lua 代码区域（使用 `SelectableText` 可复制） |
| ActionButtons | 「执行」「重新生成」双按钮 |

### 步骤颜色映射

| 阶段 | 色码 | 说明 |
|------|------|------|
| 前戏/开始 | `#b2f2bb` 绿色 | 舒适阶段 |
| 等待/间歇 | `#fff3bf` 黄色 | 等待用户操作 |
| 渐进/积累 | `#ffd8a8` 橙色 | 强度上升中 |
| 高潮/峰值 | `#ffc9c9` 红色 | ⛰️ 标记 |
| 收尾/结束 | `#b2f2bb` 绿色 | 安全收尾 |

```
│  │    赎罪计时           │  │  ← 阶段名称（大号）

│  └───────────────────────┘  │

│  整体进度                    │
│  ▓▓▓▓▓▓▓░░░░░  60%         │  ← 进度条

│  ┌───────┐ ┌───────┐      │
│  │🌀60% │ │🔒 ON  │      │  ← 玩具状态卡片
│  ├───────┤ ├───────┤      │     · 2x2 网格
│  │💧监测中│ │⚡就绪│      │
│  └───────┘ └───────┘      │
│                              │
│  执行日志                    │
│  ┌───────────────────────┐  │
│  │ [阶段3] 赎罪计时开始  │  │  ← 日志滚动区域
│  │ 维持气压！倒计时 18s  │  │
│  └───────────────────────┘  │
│                              │
│  ┌───────────────────────┐  │
│  │     🛑  紧急停止       │  │  ← 紧急停止按钮
│  │     (双击确认)         │  │     · 红色全宽按钮
│  └───────────────────────┘  │     · 需二次确认弹窗
└─────────────────────────────┘
```

### 组件清单

| 组件 | 说明 |
|------|------|
| StageDisplay | 当前阶段展示卡片（大号数字+名称+描述） |
| ProgressBar | 整体执行进度条 |
| ToyStatusGrid | 2x2 网格，每个显示玩具名+当前强度/状态 |
| LogView | 滚动的执行日志（显示 print 输出） |
| EmergencyStop | 红色大按钮 + 二次确认弹窗 |

### 交互细节

| 交互 | 行为 |
|------|------|
| 阶段切换 | 当 Lua 脚本执行到 `print("[阶段N] ...")` 时自动更新阶段显示 |
| 倒计时更新 | 每秒更新展示剩余秒数 |
| 进度条 | 根据总时长计算百分比 |
| 紧急停止 | 点击→弹出 Dialog「确认停止？所有玩具将立即关闭」→确认→调用所有 `emergencyStop()` |
| 执行完成 | 自动跳转回首页，显示「完成」提示 |

## 7. 设计规范

### 配色

| 用途 | 色值 |
|------|------|
| 主色（按钮/标题） | `#4a9eed` 蓝色 |
| 成功/连接 | `#22c55e` 绿色 |
| 警告/等待 | `#f59e0b` 黄色 |
| 危险/停止 | `#ef4444` 红色 |
| 特殊/Agent | `#8b5cf6` 紫色 |
| 背景 | `#ffffff` |
| 卡片背景 | `#f8f9fa` |

### 字体

| 层级 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 20 | Bold |
| 方案名称 | 18 | Bold |
| 阶段名称 | 24 | Bold |
| 正文 | 14 | Regular |
| 辅助/标签 | 12 | Regular |
| Lua 代码 | 10 | monospace |

### 间距

| 层级 | 值 |
|------|----|
| 页面边距 | 16px |
| 卡片内边距 | 12px |
| 组件间距 | 8px |
| 段落间距 | 16px |
